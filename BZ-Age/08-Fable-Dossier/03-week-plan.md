# The Fable Window — Plan and Rationale

**Written:** July 2, 2026 (Fable). Governs roughly five days of Fable access.
**Mandate:** `04-Working-Notes/fable-master-prompt.md`. This file records how that mandate is being
spent and why — including where the plan departs from the prompt.

## The organizing principle

Rank every task by **(how much it needs Fable) × (how hard it is to retrofit later)**. Spend the
window at the top of that ranking; push everything else down to Opus with a spec. Two corollaries:

1. The scarcest asset is not Fable's output — it's Fable's *taste, codified into rubrics with
   examples*. A Voice Bible Opus can apply is worth more than ten beats Opus could have generated
   from a taxonomy.
2. Volume generation (beat menus, glimpse candidates, recency searches) is Fable-*good* but not
   Fable-*only*. Do it at favorites-depth, hand over the generator, let Opus extend.

## The triage the master prompt asked for

**Fable-only (spend the window here):**
- The ear: distinguishing the earned Carlin move from the reflexive tic (Voice Bible)
- The judgment: the brutal architecture critique, biggest-problem-first
- Seed-level design that can't be retrofitted: the recurring cast, the peak emotional beats,
  the bookends, the reception spine
- Connection-making across distance (the why-here argument, the glimpse-bank *story-work* column)
- Taking and defending positions (mission question, homage question) → decision log

**Opus-able (spec it, don't do it):**
- The P3–P9 draft-04 surgical pass — fully specified by the fleet findings; becomes the
  **pilot test of the whole handoff model** (see below)
- Audiobook reassembly from draft-04 files
- Recency-check *searching* (cheap research agents with web access; Fable only triages results)
- Extending beat menus / glimpse bank once the taxonomy and exemplars exist
- All future fact-fleet mechanics (the journaled-workflow patterns are already proven and documented)

## The pilot: test the handoff while the brain is still here

The pending P3–P9 pass moves to Opus *deliberately*, not by neglect. Fable writes
`14-draft-04-execution-spec.md` (per-part ordered checklist from the banked fleet findings, with
example sentences marked where a fix needs new prose in-voice). The writer runs Opus against it
mid-week. Fable reviews Opus's output against the spec **before the window closes** — so we learn
whether the specs are precise enough while corrections are still cheap. If Opus holds the voice on
surgical work, the handoff model is validated; if it doesn't, the Operating Manual gets rewritten
around what actually failed.

## Departures from the master prompt (argued, per its own rules)

1. **Added deliverable #0: the Opus Operating Manual.** The prompt asks for seeds and machinery but
   never names the document the successor boots from. Without per-task recipes and a do-not list,
   the dossier is a library with no librarian.
2. **Added the decision log.** The prompt asks Fable to take positions; positions taken in chat
   evaporate. Logged decisions with reversal costs are what stop Opus from relitigating.
3. **Merged A+B and D+E.** Same devices, different zoom levels. Treating them as four commissions
   would produce padding.
4. **Demoted Commission I to a delegated, rolling task.** Recency checking is the most Opus-able
   commission on the list — web search is not a Fable skill worth paying Fable prices for.
5. **Do not fresh-boot Fable sessions by pasting the prompt.** Continued sessions keep the loaded
   project context and the prompt cache warm; a fresh boot pays for re-reading the repo. The prompt
   file is the standing mandate; this dossier's index is the cold-start re-entry point if a fresh
   session is ever unavoidable.

## Credit posture

This phase is read-and-write heavy, fleet-light — structurally much cheaper than the audit fleets
that preceded it. The only agent spend: small research workflows for the glimpse bank and recency
flags, routed to cheaper model tiers. Fable tokens go to judgment and to writing this dossier.
